# Contribution Guidelines

Thank you for your interest in contributing to the Microsoft Fabric project! We welcome contributions from the community and appreciate your efforts in helping us improve. Please follow these guidelines to ensure that your contributions are in line with the goals of the project.

## Types of Contributions
- **Bug Reports**: If you find a bug, please create an issue in the repository with a clear description of the problem, steps to reproduce it, and any relevant screenshots or logs.
- **Feature Requests**: If you have a suggestion for a new feature, feel free to open an issue to discuss it with the maintainers before starting work on it.
- **Code Contributions**: You're welcome to submit code changes via pull requests. Please follow the coding standards and ensure that your code is well-documented.

## Issue Reporting Procedures
1. **Check Existing Issues**: Before creating a new issue, ensure that it has not already been reported by searching through existing issues.
2. **Create a New Issue**: If your issue is unique, create a new issue using the template provided. Make sure to fill out all fields.
3. **Provide Detailed Information**: Include details such as the environment setup, steps to reproduce, and any error messages you encountered. The more information you provide, the easier it will be for us to assist you.

## Code of Conduct
We expect all contributors to adhere to our Code of Conduct. Be respectful of others, inclusive, and constructive in your conversations. Any form of harassment or discrimination will not be tolerated. For more details, please refer to the full Code of Conduct in this repository.

## Acknowledgment
We appreciate every contribution and thank you for helping us improve Microsoft Fabric! If you have any questions, feel free to reach out to the maintainers.
